**Testing strategy: Use sequence-controlled stubs.** To test branching, the executor must control the outcome sequence for specific nodes. Create a helper class in tests/test_engine.py:

```python
class ScriptedNode(Node):
    """Returns outcomes from a pre-defined sequence, then falls back to a default."""
    def __init__(self, outcomes: list[Outcome], default: Outcome):
        self._outcomes = list(outcomes)
        self._default = default
    def execute(self, job: JobState) -> Outcome:
        return self._outcomes.pop(0) if self._outcomes else self._default
```

This lets tests inject specific outcome sequences by monkey-patching `engine._registry[node_name]`.
