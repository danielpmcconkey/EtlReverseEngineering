Trailer handling in Append mode:
  - Deterministic trailer (row_count + date): Proofmark compares trailer normally. Cumulative row_count must match.
  - Non-deterministic trailer ({timestamp}): Proofmark uses trailer_match: skip
  - The framework strips the prior date's trailer before union and re-appends after writing cumulative data

Dependency chain (NOT data dependencies -- safety ordering only):
  DailyTransactionSummary -> DailyTransactionVolume -> MonthlyTransactionTrend
  All three source from datalake.transactions / datalake.branches, NOT from each other's output.
  Execute in order as a safety measure. Can technically run in parallel.

CRLF line ending:
  DailyTransactionVolume uses "lineEnding": "CRLF" -- must match V1 exactly.

Append mode retry protocol (same as Plan 01):
  1. Check which dates succeeded vs failed
  2. If failed date has stale output, delete it
  3. Reset failed tasks to Pending
  4. Nuclear option: delete entire RE output dir for that job, reset ALL tasks
