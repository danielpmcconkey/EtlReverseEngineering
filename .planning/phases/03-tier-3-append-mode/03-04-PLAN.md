<!-- Parquet Append mode specifics -->

ParquetFileWriter Append mode works identically to CSV:
  - Reads prior partition's parquet directory via DataFrame.FromParquet
  - Drops etl_effective_date column
  - Unions with current date's data
  - Re-injects etl_effective_date
  - Writes combined result

RE job conf pattern for Parquet Append:
  {
    "type": "ParquetFileWriter",
    "source": "result",
    "writeMode": "Append",
    "numParts": 2,
    "outputDirectory": "{ETL_RE_OUTPUT}",
    "jobDirName": "...",
    "fileName": "...",          // directory name, no extension
    "outputTableDirName": "..."
  }

Proofmark config for Parquet (no trailer handling):
  comparison_target: JobName
  reader: parquet

Proofmark queue for Parquet (directory paths, no file extension):
  lhs_path: '{ETL_ROOT}/Output/curated/{jobDirName}/{tableDirName}/{date}/{fileName}'
  rhs_path: '{ETL_RE_OUTPUT}/{jobDirName}/{tableDirName}/{date}/{fileName}'

Empty-vs-empty dates: Some Parquet Append jobs may have dates with no new data.
  Unlike Overwrite (which can produce truly empty dirs), Append should always have data
  after the first date because it carries forward all prior data. However, verify behavior.
