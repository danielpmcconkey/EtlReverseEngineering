Append mode key difference from Overwrite:
  - RE job conf MUST have "writeMode": "Append" in the writer module
  - All 92 dates MUST be queued in a single INSERT (framework claims entire batch via advisory lock)
  - Execution order is load-bearing -- framework sorts by effective_date automatically
  - Retry protocol: inspect + clean stale RE output BEFORE resetting tasks to Pending

Append mode retry protocol (different from Overwrite):
  1. Check which dates succeeded: SELECT effective_date, status FROM control.task_queue WHERE job_name = 'JobName_RE' ORDER BY effective_date
  2. If failed date has stale output, delete it: rm -rf {ETL_RE_OUTPUT}/{jobDirName}/{tableDirName}/{failed_date}/
  3. Reset failed tasks: UPDATE control.task_queue SET status = 'Pending', error_message = NULL WHERE job_name = 'JobName_RE' AND status = 'Failed'
  4. Nuclear option (full re-run): rm -rf {ETL_RE_OUTPUT}/{jobDirName}/{tableDirName}/ then reset ALL tasks
